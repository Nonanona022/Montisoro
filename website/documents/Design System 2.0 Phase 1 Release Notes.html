02Release Notes

Toegevoegd

  • Lead Paragraph System — intro-alinea's (17–21px, lh 1.72, 60ch, text-wrap:pretty) op alle pagina's
  • Meaningful Numbers (.ds-num) — Playfair-oranje kerncijfer in lopende tekst → Home, About (NL+EN)
  • Vertical Rhythm Tokens — gedeelde spatiëringsschaal
  • CTA Microcopy (.cta-note) — geruststelling onder primaire CTA → Home, About, Aanpak, Contact (NL+EN)
  • Micro-interactions — uniforme knop-hover/focus, reduced-motion-safe, op alle .btn
  • Documentatie — Guidelines + Decision Log + deze Release Notes

Verbeterd

  • CTA-microcopy ontdaan van redundantie ("vrijblijvend" + "geen verplichting" → één feitelijke regel)
  • Scanbaarheid van lange tekst (lead-paragraph-instappunten)
  • Premium-afwerking via consistente, ingetogen knop-feedback

Verwijderd / opgeruimd

  • Inline beta-stijl op Home (style#ds2-beta) vervangen door gedeelde ds2.css

Opgeloste bugs / regressies (in deze release-cyclus)

  • 404-pagina miste ds2 → toegevoegd (consistency-regressie)
  • forms.css ontbrak op calculator-en / fit-check-en → toegevoegd (parity)
  • iOS auto-zoom op formuliervelden → 16px-fix sitebreed
  • Marquee-logo's laadden niet op mobiel → eager loading
  • Taalknop sprong naar contact → bidirectionele PAGE_MAP

03Technical Debt (gedocumenteerd, niet uitgevoerd)

  • Fase 1-restant — bewust beperkt: .ds-num/.cta-note staan enkel waar de impactanalyse groen was. Geen schuld, maar een keuze (kwaliteit boven uniformiteit) — gedocumenteerd voor toekomstige ontwikkelaars.
  • Dode CSS: .hs-reassure in contact.html is ongebruikt (cta-note vervangt de functie). Opruimen kan, lage prioriteit.
  • Rhythm-tokens: nu licht toegepast; een volledige token-schaal (sectie/blok/element) kan in Fase 2 formeler.
  • 'unsafe-inline' CSP: bewuste tradeoff (geen build-step); injectie-vector al afgedekt via esc().
  • Server-side PDF: Chromium faalt op Netlify-timeout; opgevangen met rijke e-mail + browser-print. Externe PDF-service = mogelijke toekomst.

04Freeze-status

Design System 2.0 — Phase 1 is vanaf nu "Feature Frozen".

Geen wijzigingen meer aan Fase 1, behalve: bugfix, security, of regressie. Elke nieuwe verbetering of nieuw component gaat naar de Phase 2-backlog en volgt de beta → goedkeuring → uitrol → post-audit-flow.
