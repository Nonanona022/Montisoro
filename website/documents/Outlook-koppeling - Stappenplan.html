De website leest de agenda van de baas (welke slots zijn al bezet?) en schrijft de nieuwe afspraak erin terug. Eén agenda = geen dubbelboeking. De bevestigingsmail gaat los daarvan uit via hello@montisoro.com — dat blijft ongewijzigd.

01

Eenmalig instellen bij Microsoft Jullie IT / M365-beheer

In het Azure-/Entra-portaal (portal.azure.com → "App-registraties"). Een klus van ±10 minuten. Resultaat: drie sleutels die je aan mij bezorgt.

Nieuwe app registreren

Naam bv. "Montisoro Boekingen". Account-type: Single tenant (alleen jullie organisatie). Geen redirect-URI nodig.

Clientgeheim aanmaken

Onder Certificaten & geheimen → Nieuw clientgeheim. Noteer meteen de waarde (die is maar één keer zichtbaar).

Rechten geven (Application permissions)

Onder API-machtigingen → Microsoft Graph → Application permissions:

  • Calendars.ReadWrite — vrij/bezet lezen + afspraak aanmaken
  • OnlineMeetings.ReadWrite — Teams-link genereren

Daarna op "Beheerderstoestemming verlenen" klikken (vereist een admin).

De agenda kiezen

Bepaal welk postvak de afspraken bevat — de Outlook-agenda van de baas, bv. laurence@montisoro.com. Dít adres geef je me door. (Een gedeelde mailbox zoals afspraken@montisoro.com kan ook — vaak netter.)

De drie sleutels + het agenda-adres aan mij bezorgen

Stuur ze veilig door. Ik zet ze als environment variables op de server — ze komen nooit in de website-code te staan.

02

De vier waarden die ik nodig heb

Zodra je deze hebt, kan ik bouwen. Ze worden veilig op de server bewaard (Netlify environment variables).

SleutelWat het is · waar je het vindt
MS_TENANT_IDDirectory (tenant) ID — op de overzichtspagina van de app-registratie
MS_CLIENT_IDApplication (client) ID — idem, overzichtspagina
MS_CLIENT_SECRETDe geheime waarde uit stap 2 (Certificaten & geheimen)
MS_CALENDAR_USERHet e-mailadres van de agenda, bv. laurence@montisoro.com

Veiligheid

Deze waarden geven toegang tot de agenda — behandel het clientgeheim als een wachtwoord. Ze staan straks alléén server-side; de bezoeker van de website ziet ze nooit.

03

Wat ik daarna bouw Ik

De boekingsflow heeft al twee "naden" klaarliggen — nu nog met gesimuleerde data. Ik vervang die door echte Graph-calls.

Een Graph-helper in de backend

Een nieuw bestandje dat met de vier sleutels een toegangstoken ophaalt bij Microsoft (client-credentials). Eén centrale plek; de rest gebruikt het.

Beschikbaarheid live maken — slotsFor()

Vandaag verzint de website welke slots bezet zijn. Ik vervang dat door Graph getSchedule (vrij/bezet van de agenda). Het vaste weekschema mínus wat al in Outlook staat = de echte vrije slots. Bezet valt automatisch weg.

De afspraak echt aanmaken — submitBooking()

Bij "Bevestig afspraak" maakt de backend een echt agenda-item aan in de agenda van de baas:

  • Online → isOnlineMeeting: true → Teams-link automatisch gegenereerd
  • Ter plaatse → het opgegeven adres als locatie

De bezoeker komt als genodigde op het event → krijgt zo ook de echte agenda-uitnodiging.

De bevestigingsmail vullen met de echte link

De bestaande gebrande afspraakmail krijgt nu de echte Teams-link / .ics-agendaknop — niet langer een placeholder.

Dubbelboek-slot dichttimmeren

Tussen "slot tonen" en "afspraak aanmaken" doet de backend een laatste vrij/bezet-check, zodat twee mensen niet tegelijk hetzelfde slot pakken.

04

Samen testen & live zetten

Eén online testboeking → verschijnt in de agenda mét Teams-link
Eén ter-plaatse testboeking → verschijnt met het juiste adres
Een bezet slot in Outlook → verdwijnt van de website
Bevestigingsmail komt aan via hello@montisoro.com (vereist ook de Resend-sleutels, K1)
NL én EN flow doorlopen — beide schrijven naar dezelfde agenda
!

Twee koppelingen, los van elkaar

De agenda (deze koppeling, Microsoft Graph) en de mailverzending (Resend-sleutels, "K1") staan los. Voor een volledig werkende flow zijn ze allebei nodig — maar je kunt ze in elke volgorde aanzetten.
